CANopen_BoardCmd

La funzione CANopen_BoardCmd consente di fermare e di riavviare la comunicazione sulla rete CANopen.

 

Sintassi

 

 

ret_code := CANopen_BoardCmd (ExecMode, ExecStatus, Board, Command, CfgFileName) ;

 

 

 

Parametri di ingresso

 

ExecMode (INT)             Modalità di esecuzione del comando [solo MODE_WAIT]

Board (INT)                    Scheda master su cui opera la funzione. Non significativo per ATCMControl.

Command (INT)             Indica il tipo di comando da inviare.

CfgFileName (STRING)   Indica il file che descrive la configurazione della rete CANopen. Se il

                        parametro contiene la stringa vuota viene usato il file di configurazione di

                        default. Questo parametro è significativo solo se il parametro Command

                        ha il valore cmd_START

 

Parametri di uscita

 

ExecStatus (DWORD)   Stato di esecuzione del comando in modalità MODE_NOWAIT_ACK.

 

 

Modalità di esecuzione

 

Impostato da ExecMode

 

 

Uso

 

La funzione gestisce esegue operazioni sullo stato della comunicazione in base al parametro Command che può assumere i seguenti valori :

 

Command

Valori

Descrizione

cmd_RESET

1

RESET della comunicazione CANopen

cmd_START

2

START della comunicazione CANopen

cmd_STOP

3

STOP della comunicazione CANopen

 

Il comando di RESET fa cadere ogni comunicazione CANopen, quindi i nodi remoti potranno andare nello stato di WatchDog. Per riattivare la comunicazione alla fine del comando di RESET bisogna sempre dare il comando di START.

Il comando di START ripristina la comunicazione CANopen ed effettua tutta la sequenza di “Node Bootup” impostata nel file di configurazione.

Il comando di STOP ferma ogni attività CANopen sul BUS. Non è implementato su ATCMControl.

Nello stato di STOP e durante l’operazione di RESET i nodi remoti potranno andare nello stato di WatchDog.

Valori di ritorno

 

Valore (HEX)

Descrizione

0

Funzione eseguita correttamente

  0x00350004

Tipo di comando non valido

  0x00350020

DRIVER Error durante inizializzazione interna (errore driver)

  0x00350032

DEVICE RESET timeout esecuzione comando (errore driver)

  0x00350033

DEVICE COM-flag non settato (errore driver)

  0x0035003E

USER IOCTRL funzione fallita (errore driver)

  0x0035003F

Parametro utente board non valido

 

 

Esempio

 

ExecStatus : dword;

Ret : dword;

 

Ret := CANopen_BoardCmd(MODE_WAIT, ExecStatus, 0, cmd_RESET, ‘’);

 

(* Fa cadere la comunicazione CANopen. *)
