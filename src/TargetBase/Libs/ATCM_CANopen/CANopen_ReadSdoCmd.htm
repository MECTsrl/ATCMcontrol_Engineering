CANopen_ReadSdoCmd

La funzione CANopen_ReadSdoCmd consente di leggere un oggetto dell’Object Dictionary di un nodo

 

Sintassi

 

 

ret_code := CANopen_ReadSdoCmd(ExecMode, ExecStatus, Board, NodeId, Index, subIndex,

       Timeout, DataLength, Data);

 

 

 

Parametri di ingresso

 

ExecMode (INT) Modalita di esecuzione del comando [MODE_NOWAIT,

                                                MODE_WAIT, MODE_WAIT_ACCEPT, MODE_NOWAIT_ACK].

Board (INT)                    Scheda master su cui opera la funzione. Non significativo per ATCMControl.

NodeId (INT)                  Identificativo del nodo

Index (WORD)               indice dell’oggetto dell’object dictionary da leggere

subIndex (INT)               sottoindice dell’oggetto dell’object dictionary da leggere

Timeout (INT)                Timeout in ms (=0 nessun timeout). Non significativo per ATCMControl

 

Parametri di uscita

 

ExecStatus (DWORD)           Stato di esecuzione del comando in modalità MODE_NOWAIT_ACK.

DataLength (INT)                    lunghezza dati ricevuti

Data (ANY_ELEMENTARY)   dati ricevuti

 

Modalità di esecuzione

 

MODE_WAIT:

La funzione rimane in attesa di una risposta dal nodo remoto, salvo il caso in cui è stato specificato un tempo di timeout; in questo caso, se non dovesse ricevere alcuna risposta, ritorna l'errore di timeout.

MODE_NOWAIT_ACK:

La funzione non rimane in attesa di una risposta dal nodo remoto. La prima volta effettua la richiesta, mentre le volte successive controlla che sia arrivata la risposta, altrimenti rimane in attesa od esce con errore di timeout.

 

Per ATCMControl la funzione può essere eseguita soltanto in modalità MODE_NOWAIT_ACK.

 

Uso

 

La funzione permette di leggere le informazioni contenute in un oggetto dell’Object Dictionary di un nodo. L'oggetto è specificato tramite i parametri Index e subIndex.

Nel parametro Index bisogna scrivere il valore esadecimale dell’indice dell'oggetto, mentre il suo sub-indice deve essere specificato nel parametro subIndex.

La funzione ritorna in uscita nel parametro DataLength la lunghezza dei dati dell’oggetto e nel parametro Data i dati dell'oggetto.

 

Valori di ritorno

Valore (HEX)

Descrizione

0

Funzione eseguita correttamente

0x00350001

Timeout invio messaggio (solo modalità WAIT)

0x00350002

Timeoout ricezione messaggio (solo modalità WAIT)

0x00350003

Il servizio è stato rifiutato dal nodo con il servizio di Abort SDO. La causa potrebbe essere dovuto al parametro Index e subIndex non validi oppure non si hanno i diritti di accesso al nodo.

I dati in uscita alla funzione contengono il codice relativo a Abort SDO.

0x00350004

Comando errato

0x00350005

Lista coda comandi piena (modalta’ WAIT)

0x00350006

Chip CAN della scheda master nello stato di OFFLINE

0x00350007

Il buffer di ricezione ha superato i limiti.

0x00350008

I dati di protocollo frammentati superano i limiti del buffer di ricezione.

0x00350009

Il servizio precedente è ancora attivo e non confermato all'applicazione.

0x0035000A

Errore di sequenza nella frammentazione del protocollo. La richiesta è abortita.

0x0035000B

Nessuna risposta dal nodo o nodo non presente

0x0035000C

Il nodo non si trova nello stato operazionale e l'accesso viene negato. Il canale SDO è usato al momento dalla scheda master per la configurazione del nodo nella fase di avvio. Riprovare a richiedere il servizio.

 

 

ATCMControl supporta solo gli SDO di tipo expedited (fino a 4 byte di dati).

 

Esempio

 

In questo esempio viene letto l’oggetto di indice 16#2010, subindice 2 del nodo 6.

 

VAR

    Datalen : int;

    Data : int;

    RetCode : dword;

    ReadExecStatus : dword;

    ReadCompletedWithSuccess : bool := false;

    ReadCompletedWithError : bool := false;

    SdoUnderWay : bool := true;

END_VAR

 

IF SdoUnderWay THEN

    RetCode := CANopen_ReadSdoCmd(MODE_NOWAIT_ACK, ReadExecStatus, 0, 6,

                                                  16#2010, 2, 0, Datalen, Data);

    IF ReadExecStatus = 16#80000000 THEN

        (* lettura SDO in corso *)

        ReadCompletedWithSuccess := false;

        ReadCompletedWithError := false;

    ELSIF ReadExecStatus = 0 THEN

        (* lettura SDO completata con successo;

           adesso la variabile Data contiene i dati letti tramite SDO *)

        ReadCompletedWithSuccess := true;

        SdoUnderWay := false;

    ELSE

        ReadCompletedWithError := true; (* lettura SDO completata con errore *)

        SdoUnderWay := false;

    END_IF;

END_IF;
