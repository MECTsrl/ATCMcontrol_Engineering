XMLReadAttributeEx

La funzione XMLReadAttributeEx legge uno o più dati dagli attributi di un file XML

 

 

Sintassi

 

 

ret_code := XMLReadAttributeEx (XMLhandle, XmlPath, XmlAttribName, Mode, VarName) ;

 

 

 

Parametri di ingresso

 

            XMLhandle(DINT)                       identificatore univoco del file XML.

            XmlPath (STRING)                     path XML dell’elemento da cui leggere I dati

            XmlAttribName (STRING)           nome dell’attributo XML contenente il dato

            Mode (WORD)                           modalità di esecuzione della funzione

            VarName (STRING)                    variabile PLC in cui scrivere i dati letti da file XML

 

Parametri di uscita

 

 

Modalità di esecuzione

 

Wait

 

 

Uso

 

Tale funzione permette di leggere dati dagli attributi degli elementi di un file XML e di scriverli  sulla variabile di nome VarName, definita all’interno del programma PLC.

Il file XML da cui leggere i dati è identificato dal parametro XMLhandle: tale parametro viene ritornato dalla funzione XMLOpen alla apertura del file.

Il parametro XmlPath consente di selezionare l’elemento all’interno del file XML da cui leggere i dati: se l’elemento identificato dal parametro XmlPath non esiste nel file XML allora la funzione ritorna errore.

Il parametro XmlAttribName consente di selezionare l’attributo da cui leggere il dato: questo parametro è significativo solo se VarName è una variabile semplice e non una struttura.

 

A seconda del tipo della variabile VarName questa funzione consente di leggere un singolo valore scalare oppure una struttura che può essere anche annidata. All’interno di una struttura possono essere presenti anche array di strutture a patto di rispettare questa limitazione: il valore più basso dell’indice dell’array deve valere 1, p.es.  StructArray : ARRAY[1..10] OF Mystruct;

 

 

La funzione ricerca all’interno dei simboli la variabile di nome VarName e ne determina il tipo. Il tipo della variabile VarName può essere uno di questi:

 

·         ANY_ELEMENTARY.  In questo caso la funzione legge il dato dall’attributo XmlAttribName dell’elemento identificato dal parametro XmlPath; la funzione scrive il dato direttamente nella variabile VarName. Il tipo della variabile VarName deve essere compatibile col tipo del dato associato all’attributo XmlAttribName dell’elemento identificato dal parametro XmlPath.

·         STRUCT. La funzione crea la lista delle variabili di tipo ANY_ELEMENTARY che sono contenute all’interno della struttura e legge i dati dagli attributi degli elementi che hanno lo stesso nome della variabile corrispondente. Il tipo delle variabili deve essere compatibile col tipo dei dati associati all’attributo corrispondente. In questo caso il parametro XmlAttribName non è significativo. Se all’interno della struttura ci sono variabili di tipo STRUCT, allora la funzione itera il processo di lettura su tale struttura. Se all’interno della struttura è presente un array di strutture la funzione itera il processo di lettura sull’array di strutture; se però il numero di elementi dell’array presenti su file (N1) è inferiore al numero di elementi dichiarati a livello PLC (N2) allora vengono caricati nell’array solo i primi N1 elementi.

Nota bene: la variabile VarName NON può essere un elemento di un array, come p.es. ‘ArrName[2]’.

 

La variabile VarName può essere una delle variabili seguenti:

·         Una variabile globale: in questo caso basta indicare nel parametro VarName il nome della variabile, p.es ‘GlobalVarName’

·         Una variabile definita all’interno di un programma: in questo caso occorre scrivere nel parametro  VarName il nome dell’istanza del programma seguito dal nome della variabile, p.es.  ‘ProgramInstanceName.VarName’

·         Una variabile definita all’interno di un function block: in questo caso occorre scrivere nel parametro  VarName il nome dell’istanza del programma seguito dal nome dell’istanza del function block seguito dal nome della variabile, p.es. ‘ProgramInstanceName.FunctionBlockInstanceName.VarName’

 

Il parametro Mode è una maschera di bit con il seguente significato:

 

Bit

Descrizione e valori possibili

0

se impostato a 1: se nel file XML mancano uno o più attributi tra quelli che si desidera leggere la funzione ritorna comunque con successo

se impostato a 0: se nel file XML mancano uno o più attributi tra quelli che si desidera leggere la funzione ritorna con errore

1..15

Non utilizzati (impostare a 0)

 

 

 

Valori di ritorno

 

Nella successiva tabella sono riportati i possibili valori assunti dalla variabile ret_code

           

ret_code (HEX)

Descrizione

0x00000000

Funzione eseguita senza errori

0x00380002

Il parametro XMLhandle non è valido

0x00380005

Errore di allocazione di memoria

0x0038000A

L’elemento identificato da XmlPath non esiste nel file XML

0x0038000C

Il tipo dei dati presenti nel file XML non è compatibile con quello della variabile PLC corrispondente

0x0038000E

La variabile VarName non esiste all’interno del programma PLC

0x0038000F

Errore interno

0x00380011

Attributo non presente nel file XML oppure attributo vuoto

 

 

 

 

Esempio

 

Ret       : dword;

XMLhandle : dint;

IntVar    : int;

…

(* apre il file SSD\osai\file.xml in lettura, legge il dato associato all’attributo attrib1 del secondo elemento elem e lo copia nella variabile globale IntVar, e infine lo chiude. Il path XML '/top/elem[2]' identifica il secondo elem quindi, in questo esempio, nella variabile IntVar verrà scritto il valore 20. *)

 

Ret := XMLOpen (‘\SSD\osai\file.xml’, XML_FMODE_READ, XMLhandle) ;

Ret := XMLReadAttributeEx(XMLhandle,'/top/elem[2]','attrib1', 0, 'IntVar');

Ret := XMLClose(XMLhandle);

 

-

Questo è il contenuto del file SSD\osai\file.xml:

<?xml version="1.0"?>

<top>

  <elem attrib1="10" attrib2="a" />

  <elem attrib1="20" attrib2="b" />

  <elem attrib1="30" attrib2="c" />

</top>
