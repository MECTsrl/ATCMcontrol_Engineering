LoadLibrary

La funzione LoadLibrary consente di caricare una dll scritta dall’utente nello spazio di indirizzamento del processo chiamante.

 

Sintassi

 

 

ret_code := LoadLibrary(LibraryName, StartFunctionOrd, EndFunctionOrd, StartFBOrd, EndFBOrd);

 

 

 

Parametri di ingresso

 

LibraryName (STRING)               Nome della dll da caricare, con tutto il path

StartFunctionOrd (INT)                Ordinale della funzione della libreria con ordinale più piccolo

EndFunctionOrd (INT)                 Ordinale della funzione della libreria con ordinale più grande

StartFBOrd (INT)                        Ordinale del function block della libreria con ordinale più piccolo

EndFBOrd (INT)                         Ordinale del function block della libreria con ordinale più grande

 

 

Uso

 

La funzione permette di caricare una dll scritta dall’utente nello spazio di indirizzamento del processo chiamante; dopo il caricamento della dll, la applicazione PLC potrà utilizzare le funzioni e i function block appartenenti alla dll.

Il parametro LibraryName contiene il nome, completo di path, della dll da caricare.

A ogni funzione appartenente alla dll deve essere assegnato un numero ordinale univoco, che deve corrispondere all’ordinale assegnato alla funzione nel file .def della dll.

I numeri ordinali delle funzioni presenti nella dll devono essere compresi all’interno del range specificato dai parametri StartFunctionOrd e EndFunctionOrd.

Gli ordinali delle funzioni della libreria devono essere comunque compresi all’interno del range 1000 – 1499.

Se nella libreria non è presente nessuna funzione occorre assegnare a entrambi i parametri StartFunctionOrd e EndFunctionOrd il valore 0.

A ogni function block appartenente alla dll deve essere assegnato un numero ordinale univoco, che deve corrispondere all’ordinale assegnato al function block nel file .def della dll.

I numeri ordinali dei function block presenti nella dll devono essere compresi all’interno del range specificato dai parametri StartFBOrd e EndFBOrd.

Gli ordinali dei function block della libreria devono essere comunque compresi all’interno del range 500 – 999.

Se nella libreria non è presente nessun function block occorre assegnare a entrambi i parametri StartFBOrd e EndFBOrd il valore 0.

Se necessario è possibile definire all’interno della dll una funzione UserInitLibrary, che inizializza la dll: la funzione UserInitLibrary deve avere ordinale 1 nel file .def della dll. La funzione LoadLibrary verifica se nella dll è presente una funzione con ordinale 1 e se è presente la chiama. La funzione UserInitLibrary deve avere questo prototipo:

BOOL  UserInitLibrary(dword *pErr);

In caso di insuccesso la UserInitLibrary ritorna FALSE (0); in questo caso il codice dell’errore ritornato dalla UserInitLibrary nel parametro pErr viene a sua volta ritornato come valore di ritorno della funzione LoadLibrary.

Se necessario è possibile definire all’interno della dll una funzione UserTerminateLibrary, che libera tutte le risorse eventualmente occupate dalla dll: la funzione UserTerminateLibrary deve avere ordinale 2 nel file .def della dll. La funzione LoadLibrary verifica se nella dll è presente una funzione con ordinale 2; se è presente la funzione UserTerminateLibrary verrà chiamate in questi casi:

·         alla chiamata della FreeLibrary, funzione che scarica la dll.

·         allo scaricamento su target di una nuova applicazione PLC

La funzione UserTerminateLibrary deve avere questo prototipo:

BOOL UserTerminateLibrary();

 

E’ possibile caricare diverse dll utente: il numero massimo di dll che possono essere contemporaneamente caricate è pari a 8.

E’ però necessario rispettare questo vincolo: i range degli ordinali delle funzioni delle diverse dll utente non si devono mai sovrapporre. Un vincolo simile esiste anche per i range degli ordinali dei function block.

 

Valori di ritorno

Valore (HEX)

Descrizione

0

Funzione eseguita correttamente

0x0015009B

Parametro StartFunctionOrd o parametro EndFunctionOrd fuori dal range 1000 – 1499

0x0015009C

Parametro StartFBOrd o parametro EndFBOrd fuori dal range 500 – 999

0x0015009D

Questa dll è già stata caricata

0x0015009E

Il range degli ordinali delle funzioni si sovrappone con quello di un’altra dll utente precedentemente caricata.

0x0015009F

Il range degli ordinali dei function block si sovrappone con quello di un’altra dll utente precedentemente caricata.

0x001500A0

Errore nel caricamento della dll

0x001500A1

E’ stato superato il numero massimo di dll utente contemporaneamente caricate

Errori definiti dall’utente

Se la funzione UserInitLibrary è presente e dà errore, allora questo errore viene riportato nel valore di ritorno della LoadLibrary

 

 

 

Esempio

 

In questo esempio viene caricata la dll '\SSD\UserDll.dll'.

Le funzioni della dll hanno ordinali nel range 1010 – 1020; nella dll non sono presenti function block.

 

VAR

    RetCode : dword;

END_VAR

 

RetCode := LoadLibrary('\SSD\UserDll2.dll', 1010, 1020,  0, 0);
