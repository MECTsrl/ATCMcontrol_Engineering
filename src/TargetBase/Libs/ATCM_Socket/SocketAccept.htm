SocketAccept

La funzione SocketAccept accetta una richiesta di connessione in ingresso.

 

Sintassi

 

 

ret_code := SocketAccept (ListeningSocket, AcceptSocket) ;

 

 

Parametri di ingresso

 

ListeningSocket (UDINT)            Socket già in ascolto di richieste di connessione

 

Parametri di uscita

 

AcceptSocket (UDINT)               Socket creato dalla funzione che sarà usato per gestire la     

                                                connessione

 

Uso

 

La funzione SocketAccept permette di accettare una richiesta di connessione in ingresso.

In particolare la funzione scoda la prima richiesta di connessione presente nella coda delle richieste di connessione del socket ListeningSocket e crea il nuovo socket AcceptSocket; il socket AcceptSocket potrà essere usato per gestire la connessione, cioè per inviare e ricevere dati lungo la connessione.

Se la coda delle richieste di connessione del socket ListeningSocket è vuota allora:

  • se il socket ListeningSocket è in modalità bloccante allora la funzione SocketAccept si blocca fino a quando viene accodata una richiesta di connessione
  • se il socket ListeningSocket è in modalità non bloccante la funzione SocketAccept esce subito con errore

In ogni caso all’uscita della funzione SocketAccept il socket ListeningSocket resta in attesa di richieste di connessione.

 

La funzione SocketAccept viene utilizzata tipicamente nel caso del protocollo TCP da un server che può avere una o più richieste di connessione contemporaneamente.

Perché un server possa accettare una connessione esso deve:

  • creare un socket tramite la funzione SocketCreate
  • associarlo a un indirizzo locale tramite la funzione SocketBind
  • metterlo in ascolto di richieste di connessione tramite la funzione SocketListen
  • accettare la connessione tramite la funzione SocketAccept

Solo a questo punto è possibile inviare e ricevere dati lungo la connessione.

 

Nel caso del protocollo UDP la funzione SocketAccept non deve essere chiamata.

 

 

Valori di ritorno

Valore (HEX)

Descrizione

0

Funzione eseguita correttamente

0x0039001E

La rete non sta funzionando

0x00390008

Buffer space esaurito o troppe richieste di connessione

0x0039000D

Il socket ListeningSocket non è stato messo in ascolto di richieste di connessione tramite la funzione SocketListen

0x00390007

Il parametro ListeningSocket non è un socket valido. Probabilmente la SocketCreate o non è stata chiamata o non ha avuto successo.

0x0039000C

A causa del suo tipo,  il socket ListeningSocket non può accettare richieste di connessione

0x0039000F

Il socket ListeningSocket è in modalità non bloccante e non esistono richieste di connessione in coda

0x00390020

Errore generico; questo errore non dovrebbe mai verificarsi

 

     

           

Esempio

 

VAR

create_res, bind_res, listen_res:  DWORD;

Socket : UDINT;

AcceptSocket : UDINT;

SocketAddress : SocketAddressType;

 

END_VAR

 

create_res := SocketCreate(SCK_AF_IPv4, SCK_PROTO_TCP, Socket);

 

if (create_res = 0) then

   SocketAddress.PortNumber := 34999;

   SocketAddress.Address := '';

   bind_res := SocketBind(Socket, SocketAddress);

   if (bind_res = 0) then

      listen_res := SocketListen(Socket, 10);

      if (listen_res = 0) then

         accept_res := SocketAccept(Socket, AcceptSocket);

      end_if;

   end_if;

end_if;
